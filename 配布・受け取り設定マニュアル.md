# すれっしゅ GitHubリンク受け取り設定マニュアル

このマニュアルは、Chrome Web Store には公開せず、GitHubの公開リンクから **すれっしゅ** を受け取って使うための手順です。

最終更新日: 2026-06-28

---

## 1. 受け取り用リンク

GitHub公開ページ:

```text
https://github.com/h11031103h/threads-export-extension
```

直接ZIPをダウンロードするリンク:

```text
https://github.com/h11031103h/threads-export-extension/archive/refs/heads/main.zip
```

通常は、上の **GitHub公開ページ** を案内してください。受け取り側がGitHubページを開いて、そこからZIPをダウンロードします。

---

## 2. 受け取り側への案内文

そのまま送れる案内文です。

```text
すれっしゅの受け取りリンクです。

https://github.com/h11031103h/threads-export-extension

使い方:
1. GitHubページを開く
2. 緑色の「Code」ボタンを押す
3. 「Download ZIP」を押してダウンロード
4. ZIPを解凍する
5. 解凍したフォルダを削除しない場所に置く
6. Chromeで chrome://extensions を開く
7. 右上の「デベロッパーモード」をONにする
8. 「パッケージ化されていない拡張機能を読み込む」を押す
9. 解凍したフォルダを選ぶ
10. Threadsを開くと、右上に「すれっしゅ」パネルが表示されます

注意:
読み込んだフォルダを削除・移動すると、拡張機能が動かなくなります。
```

---

## 3. 受け取り側のインストール手順

### 3.1 GitHubからダウンロードする

1. Chromeで以下のリンクを開く

```text
https://github.com/h11031103h/threads-export-extension
```

2. 画面右上付近の緑色の **Code** ボタンを押す
3. 表示されたメニューから **Download ZIP** を押す
4. ZIPファイルがダウンロードされる

ダウンロードされるファイル名は、通常このような名前になります。

```text
threads-export-extension-main.zip
```

### 3.2 ZIPを解凍する

1. ダウンロードしたZIPを解凍する
2. 解凍後のフォルダを確認する

解凍後のフォルダ名は、通常このようになります。

```text
threads-export-extension-main
```

3. このフォルダを、削除しない場所へ移動する

おすすめの保存場所:

```text
デスクトップ
ドキュメント
社内共有フォルダ
```

フォルダ名は `threads-export-extension-main` のままでも動きます。気になる場合は `threads-export-extension` に名前を変えても問題ありません。

### 3.3 Chromeで読み込む

1. Google Chromeを開く
2. アドレスバーに以下を入力してEnter

```text
chrome://extensions
```

3. 右上の **デベロッパーモード** をONにする
4. **パッケージ化されていない拡張機能を読み込む** を押す
5. 解凍したフォルダを選ぶ

選ぶフォルダの例:

```text
threads-export-extension-main
```

6. 一覧に **すれっしゅ** が表示され、ONになっていれば完了

`manifest.json` だけを選ぶのではなく、解凍したフォルダ自体を選んでください。

---

## 4. 使い方

1. ChromeでThreadsにログインする
2. Threadsの検索結果またはタイムラインを開く
3. 画面右上の **すれっしゅ** パネルを確認する
4. **スタート** を押す
5. 収集したい範囲までスクロールする
6. 件数が増えたら **Download CSV** を押す
7. CSVを保存する

---

## 5. 更新版を受け取る場合

配布側から「新しい版を出しました」と案内があった場合は、次の手順で更新します。

1. GitHubページを開く

```text
https://github.com/h11031103h/threads-export-extension
```

2. **Code → Download ZIP** で新しいZIPをダウンロードする
3. ZIPを解凍する
4. 古いフォルダを新しいフォルダで置き換える
5. Chromeで `chrome://extensions` を開く
6. **すれっしゅ** の **更新** ボタンを押す
7. Threadsのタブをすべて閉じる
8. Threadsを開き直す

保存場所を毎回変えると、Chromeがどのフォルダを読んでいるかわかりにくくなります。更新時は、同じ場所のフォルダを置き換える運用にしてください。

## 6. トラブルシューティング

### GitHubページが見られない

- リンクが間違っていないか確認する
- リポジトリが非公開になっている場合は、配布者にアクセス権限を確認する
- 会社や学校のネットワークでGitHubが制限されていないか確認する

### CodeボタンやDownload ZIPが見つからない

- スマホではなくPCのChromeで開く
- GitHubページ上部の緑色の **Code** ボタンを探す
- 見つからない場合は、直接ZIPリンクを開く

```text
https://github.com/h11031103h/threads-export-extension/archive/refs/heads/main.zip
```

### Chromeに読み込めない

- ZIPのまま読み込んでいないか確認する
- ZIPを解凍してから、解凍後のフォルダを選ぶ
- `manifest.json` がフォルダ直下にあるか確認する
- `chrome://extensions` でデベロッパーモードがONか確認する

### すれっしゅが表示されない

- ThreadsのURLが `threads.com` または `threads.net` か確認する
- `chrome://extensions` で **すれっしゅ** がONか確認する
- Threadsのタブを閉じて開き直す
- 読み込んだフォルダを削除・移動していないか確認する

### 「収集: 0 件」のまま増えない

- **スタート** を押しているか確認する
- ページを下へスクロールする
- 投稿が画面に読み込まれるまで少し待つ
- 「純粋ポストのみ収集」をOFFにして試す

### 更新したのに古い動作のまま

1. Threadsのタブをすべて閉じる
2. `chrome://extensions` で **すれっしゅ** の **更新** を押す
3. Chromeを再起動する
4. Threadsを開き直す

---

## 7. 注意事項

- Chrome Web Store経由ではないため、受け取り側ではデベロッパーモードを使います
- Chromeに警告が表示される場合があります
- 取得しているのは、ユーザーの画面上に表示されたThreadsの情報です
- CSVは受け取り側のPCに保存されます
- 読み込んだフォルダを削除・移動すると動かなくなります
- Threads側の画面仕様が変わると、取得精度が落ちることがあります
- 利用時はThreadsの利用規約に従ってください
